# Mission Format

`ai-capability-program/MISSION.md` grounds all teaching. Unlike a generic one-learner workspace, this program has a **cohort**: one program mission plus **one block per leader**, captured in the Week-0 intake call. Every lesson, demo, and tailored example traces back to a block here.

## Structure

```md
# Mission

## Program mission
{1–3 sentences. The concrete outcome: each leader ships a working agent + a 90-day plan. Not "understand AI".}

## Per-leader mission

### Leader N — <name> · <role>
- **Why they're here (their words):** …
- **Function & mandate:** …
- **Top 3 pain points:** …
- **Data they sit on:** …
- **Tools they live in:** …
- **What "AI-capable" looks like for them in 90 days:** …
- **Starting point (AI fluency):** none / aware / dabbled
```

## Rules

- **Concrete over abstract.** "Cut RFP turnaround from 3 days to 3 hours" beats "use AI in sales."
- **One block per leader.** The blocks drive Week-2 spec sheets, live demo choices, worked examples, and each leader's zone of proximal development.
- **Captured in intake, not guessed.** If a block is thin or a leader's "why" is unclear, that's the first thing to resolve in the intake call — ungrounded teaching feels abstract and you lose the basis for judging what to teach next.
- **Revise when reality shifts.** A leader's goal can move once they see what's possible. Update the block; don't let a stale mission steer tailoring. Cross-link any shift to a learning record.
- **Keep blocks short.** A block is a compass, not a plan. If it runs past a screen, trim it.
