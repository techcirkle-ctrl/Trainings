Agents vs. Chatbots

Preread 4 · wrapper  •  ~1 hr  •  Google — Introduction to AI Agents
Why this matters to you

An agent is the thing you will build and deploy in this program. Everything so far — the model, its limits, responsible use — leads here. This lesson draws the one line that matters: the difference between a chatbot that talks and an agent that does the work.

By the end you'll be able to spot, in your own function, a task that is agent-shaped — multi-step, repetitive, takes actions — versus one a plain chatbot already handles.

The frame

A chatbot answers. An agent is given a goal, can use tools to get things done, takes steps in a loop, and keeps track of where it is until the goal is met. It's the LLM from preread 2 with its two limits removed: tools let it act, and a task loop lets it carry work through to completion.

Carry these three away

  • Goal, not question. You hand an agent an outcome and let it work out the steps.
  • Tools are what make it act — search a system, update a record, send a message. No tools, no agent.
  • It loops. Act → check the result → decide the next step → repeat, until the goal is reached or it hits a guardrail.
