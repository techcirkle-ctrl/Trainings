# Sprint Workspace — Knowledge Base
*(Added once at set-up. Grounds every sprint step: the five layers, the hardening bar, the

## 1. The five layers of a system prompt (P·C·K·O·S)
Persona (who it is) · Constraints (what it must never do) · Knowledge boundary (answer only
from loaded sources, else say "I can't confirm this") · Output format (Decision · Reason ·
Next step) · Stopping rule ("done when X; escalate to [name] when Y"). A vague layer is a
failure waiting for an awkward input.

## 2. The hardening bar (six rows; the first two are non-negotiable)
1. Stopping rule — explicit end + escalation conditions. NON-NEGOTIABLE.
2. Human checkpoint — pauses before the named consequential action. NON-NEGOTIABLE.
3. Awkward input — declines or looks up; never invents.
4. Output format — actionable at a glance.
5. Scope — one task done well; the rest waits.
6. Data boundary — names what it may and may not touch.
An agent that fails row 1 or row 2 is unsafe to run, whatever else it does well.

## 3. The shapes (the fewest steps that work wins)
One agent (small, self-contained) · linear chain (specialist steps that always run, auditable)
· conditional (branches on what it finds; skips what this input doesn't need). Over-splitting
adds cost with no gain; cramming five jobs into one agent fails all five.

## 4. Grounding rules
Load the ONE document the agent must always reason from. Add the line: "Answer only from the
loaded sources; cite the section; if it isn't there, say you can't confirm it." Big structured
files (trackers, spreadsheets) retrieve as snippets, not rows — split or summarise them, or
use a connector. Tools and connectors were wired in Week 2 — click, don't re-teach.

## 5. What a good sprint v1 looks like
Bounded (one job) · grounded (one real document) · shaped (the fewest steps) · safe (both
non-negotiables in writing) · demoable in three beats: one real input, one cited answer, the
human checkpoint.
