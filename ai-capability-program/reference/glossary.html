AI Capability Program · Reference

Glossary

The shared vocabulary of the program. Once a term is defined here, every lesson uses it the same way. Starts with Week 1; grows each week.

AI Evolution Stack — informally: the ladder; its levels, the rungs
The program's backbone: four layers of AI capability, each adding one thing the layer below lacks — LLM → RAG → Agent → Multi-agent system. The core leadership judgement is naming the lowest layer that solves a problem: the cheapest layer that works wins. Canonical name: AI Evolution Stack; its levels are Layers. "Ladder" and "rung" are the same idea, used informally.
LLM — Large Language Model
A model that reads and writes language and reasons over what it's given, using general knowledge learned in training. The reasoning engine under everything else. Layer 1 of the AI Evolution Stack.
Context window
How much text a model can "hold in mind" at once — your prompt plus everything you've pasted in. Exceed it and the earliest content drops off.
Attention
How a model weighs which earlier words matter most when interpreting each word — for example, working out what "it" refers to in a sentence. The mechanism behind its grasp of language; you don't operate it, but it explains why context and clear phrasing change the output.
Prompt
The instruction and information you give the model. Good prompting — clear role, task, constraints, examples — is the highest-leverage skill for getting reliable output.
RAG — Retrieval-Augmented Generation
Fetching the relevant pieces of your own documents at question time and giving them to the model, so answers are grounded in your reality and can cite sources. Layer 2 of the AI Evolution Stack.
Tool use
Letting a model call external functions — search a database, send an email, hit an API — so it can act, not just talk. The capability that turns an LLM into an agent.
Agent
An LLM that uses tools, takes steps in a loop, and keeps state to reach a goal — it does multi-step work rather than just answering. Layer 3 of the AI Evolution Stack.
Multi-agent system
Several specialised agents coordinated by an orchestrator for genuinely complex workflows. The most capable and the most costly layer — used only when one agent can't cope. Layer 4 of the AI Evolution Stack.
Chatbot
An AI you ask questions and get answers from. It replies with text; it doesn't take actions or run multi-step work. The thing an agent goes beyond.
Claude Project
A workspace that gives Claude persistent instructions plus connected knowledge (and, optionally, connectors) for a specific job. The no-code build surface every leader uses in this program — no terminal, no code. Leaders assemble their agent as a Project; the trainer also runs Claude Code + MCP inside one as the program's facilitator. The build surface for every hands-on build in the program.
Token
A word-chunk — the unit an LLM reads and predicts. Text is broken into tokens; the model generates one at a time. Roughly ¾ of a word each.
Hallucination
When a model states something false with confidence. Grounding (RAG), citations, and human review are the standard guardrails — central to responsible use.
Training data
The large body of text an LLM learned its patterns from. Its knowledge is frozen at training time — blind to anything more recent or private to your business, which is the gap RAG closes.
Temperature
The creativity dial on a model's output. Low temperature → consistent, repeatable answers (it favours the most likely token); high temperature → more varied and creative, less predictable. Turn it down for factual work, up for brainstorming.
Human-in-the-loop
A person reviews or approves an AI's output before it's acted on. The core guardrail for any decision with real consequences — accountability stays human.
Responsible AI
The habits that keep an AI tool from leaking data, producing biased or false output, or acting without oversight. For a leader: a data rule, bias testing, and a human review step.
Zone of proximal development
The sweet spot just beyond what you can already do — challenging enough to grow, not so hard you stall. Each lesson is pitched here.
Agent loop
The cycle an agent repeats — think → act → observe → decide — calling tools until a stopping rule fires. It's where both an agent's power and its risk live. The "loop" in "an agent is an LLM + tools + a loop." Governed by a stopping rule and a human checkpoint.
Autonomy — the least-autonomy principle
How much an agent decides for itself versus following a fixed path. The program's rule: pick the least autonomy that still solves the problem — autonomy is a cost, not a prize, because every step of freedom is another place it can go wrong. The Week-2 sharpening of "the cheapest layer that works wins."
Workflow vs agent
Two ways to structure multi-step AI work. In a workflow you fix the path in advance (chaining, routing, parallelisation); in an agent the model chooses the path at run time. Most things called "agents" are really workflows — and that's good: cheaper, more reliable, easier to govern. Reach right (toward agent) only when the path genuinely can't be drawn in advance. See Autonomy.
ReAct
An agent pattern where the model reasons and acts in alternation — decide a step, call a tool, read the result, decide the next — so the path differs every run. The default "agent" shape. An Agent pattern; higher autonomy than any workflow.
Plan-and-Execute
An agent pattern where the model plans the whole approach up front, then carries it out — which lets a human approve the plan before it runs. An Agent pattern; pairs naturally with a human checkpoint on the plan.
Reflection
An agent pattern where the model critiques and improves its own output before finishing — a quality loop rather than new actions. The lightest agent pattern. An Agent pattern.
Tool call — tool calling
The mechanic behind tool use: the model writes a structured request for a tool; your system — not the model — decides whether to run it and hands the result back as an observation. The model only ever asks. The seam where your human checkpoint and data rules apply. The how of Tool use.
MCP — Model Context Protocol
A standard way to connect tools to an agent — think USB-C: wire a tool once and any agent can use it the same way. The plumbing under Tool use; demoed live on Claude Code in this program.
Stopping rule
How an agent knows it's done — or must escalate. Without one, an acting agent can loop or over-run. One of the two things that make an agent governed rather than just powerful; the other is the human checkpoint.
Human checkpoint
The specific point where an agent hands a decision back to a person before acting. The concrete form a human-in-the-loop takes inside an agent's design — and the governance a leader owns. The applied form of Human-in-the-loop. Tools without a checkpoint is the combination that gets a leader into trouble.
Agent lifecycle
The stages of owning an agent — design → build → test → deploy → monitor. A leader sponsors the lifecycle and owns the checkpoint; they don't operate the agent day to day. The shape of the Week-2 design doc.
