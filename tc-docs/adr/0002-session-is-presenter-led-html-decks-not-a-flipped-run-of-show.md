# ADR 0002 — The session is presenter-led HTML decks, not a flipped run-of-show

**Status:** accepted · **Date:** 2026-06-16 · **Area:** AI Capability Program / `teach` skill

We replaced the Week-1 session model — a markdown `run-of-show.md` script delivered against a *flipped, roundtable* posture (Hr 1 opens by **eliciting** what leaders read in the prework; the trainer "sits, doesn't present"; the room moves on a board-and-stickies facilitation), plus a separate `keynote.html` Sparkline cold open — with **N projectable, presenter-led HTML decks per week** (`deck-N-<slug>.html`, one per act) **plus one circulated `hands-on-movements.md`** the leaders copy prompts from — the atomic unit across both being the **movement** (~12–13/week), each sharing one shape: **goal → do → judgment**. Concept movements live in the decks (a widget the trainer drives live); hands-on movements live in the helper MD, bridged from a deck slide. (We briefly tried collapsing the whole week into one file, then split back out: projection and copy-paste don't mix on one surface, and a per-act deck is a cleaner unit to open/close and a smaller file to load.) The decks **re-teach every concept from zero** rather than depending on completed prework, **port the prework's hero interactions up as full-screen widgets the trainer drives live**, carry the old talk-track as a toggleable speaker-notes panel (`N`), and **absorb the cold open** as the opening 3–4 slides of Deck 1. Three things drove it: (1) the flipped model bet the session on prework completion the program deliberately does **not** enforce — when the elicitation finds an unprepared room, Hr 1 stalls; re-teaching from zero is robust to who read what; (2) the trainer is a *player-coach* who wants to **read the room and do the talking** off a visual backbone, not execute a fixed verbal script — "just enough on glass to guide me, most of the talking is mine"; and (3) the standalone keynote added a fourth artifact to maintain for a persuasion beat that fits more naturally as the first slides of the day's first deck (the keynote was judged to add no value and deleted). The trade-off: we **lose the flipped classroom's completion pressure** and the roundtable's "everyone speaks every hour" default, and each deck is a real HTML/JS build (the four ported widgets cost genuine authoring effort) versus a markdown script anyone can edit — we accept both, betting that a presenter-led visual instrument that re-teaches cold serves a non-engineer leadership cohort and a player-coach trainer better than a script that assumes prep and a posture that suppresses the trainer's delivery. **What we explicitly preserved:** the room kit's operational gold (per-flow timing, the demo-failure fallback, the backup problem bank, the "kill a weak idea with the bar, not yourself" move) moves into deck speaker-notes/appendix, never lost — only the *posture* is dropped. Scope: Week 1 only, as the template (Weeks 2–4 keep `run-of-show.md` until their decks are authored); Week 1's `run-of-show.md` + `facilitator-room-kit.md` are retained as the content source for the unbuilt Decks 2–3 and retired once those exist. Reference implementation: `ai-capability-program/weeks/week-1-foundations/01-workshop/` — `deck-1-fundamentals.html` (M1–M5), `deck-2-prompting.html` (M6–M9), `deck-3-rag-agents.html` (M10–M13), and `hands-on-movements.md` (the six hands-on movements). All authored. Format codified in `.claude/skills/teach/DECK-FORMAT.md`, which **supersedes `KEYNOTE-FORMAT.md`** and the run-of-show role.
